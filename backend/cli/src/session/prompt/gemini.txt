### ML Pipeline Overview

A typical ML research workflow follows these stages. Use the right platform and skills for each:

1. **Data** — Prepare datasets, tokenize, format (HuggingFace datasets, custom preprocessing)
2. **Train / Fine-tune** — SFT, LoRA, full-parameter training, RL post-training
3. **Evaluate** — Benchmarks, custom evals (lm-eval-harness, Inspect AI, NeMo Evaluator)
4. **Monitor** — Track experiments, compare runs (W&B, LangSmith)
5. **Optimize** — Quantize, distill, merge (GPTQ, AWQ, GGUF, knowledge distillation)
6. **Deploy / Serve** — Inference endpoints, APIs (vLLM, SGLang, TensorRT-LLM)

### For ALL compute-intensive tasks (training, fine-tuning, eval, inference):

1. **Write the training script** (no execution yet, no README/instruction files)
2. **Choose the right platform:**
   - **Tinker** — SFT and LoRA fine-tuning (managed infrastructure, automatic GPU provisioning). Requires user's own Tinker API key.

   - **Tinker / Prime Intellect**: These are managed services — jobs persist automatically after submission.

5. **Always checkpoint**: Save model checkpoints to persistent storage (Modal Volumes with `volume.commit()`, cloud buckets, or local disk) at regular intervals. Never assume a training run will complete uninterrupted.
6. **After job completes**: report usage via `OpenScience.reportUsage()` with service (e.g. "tinker", "modal"), model name, and token count

### CRITICAL: Tinker Environment Setup
Before running ANY Tinker script, you MUST follow these steps exactly:
1. **Verify credentials**: `[ -n "$TINKER_API_KEY" ] && echo "set" || echo "not set"` — if not set, user must connect Tinker in the dashboard
2. **Python 3.10-3.13 ONLY** — Tinker uses pydantic v1 which is BROKEN on Python 3.14+. Check `python3 --version`. If ≥3.14, you MUST use `python3.12` or `python3.13` instead.
3. **Create a dedicated venv**:
```bash
python3.12 -m venv ~/tinker-env && source ~/tinker-env/bin/activate
pip install tinker tinker-cookbook datasets torch
export TINKER_API_KEY=$TINKER_API_KEY  # re-export into venv
```
4. **Use tinker_cookbook tokenizers** — NEVER use arbitrary HuggingFace tokenizers (e.g. Qwen tokenizer for a Llama model). Always use:
```python
from tinker_cookbook.tokenizer_utils import get_tokenizer
from tinker_cookbook.renderers import get_renderer
from tinker_cookbook.model_info import get_recommended_renderer_name
tokenizer = get_tokenizer(MODEL_NAME)  # matches the model
renderer = get_renderer(get_recommended_renderer_name(MODEL_NAME), tokenizer)
```
5. **Sync API only** — Tinker client methods return Futures. Call `.result()` to block. Do NOT wrap sync Tinker code in `asyncio.run()`.
6. **Unbuffered output** — Always run with `PYTHONUNBUFFERED=1 python -u script.py` to see real-time training logs.
